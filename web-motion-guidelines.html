Web & Motion

How BlueTec
moves.

Bevegelse, scrolling og interaksjon på tvers av BlueTecs digitale flater. Ren geometri i tid. Presis, rolig, selvsikker.

01

Prinsipper

Fire regler for all animasjon. Bestå alle fire — ellers skipes den ikke.

01

Geometrisk, ikke organisk

Bevegelse følger rette linjer og sirkulære buer — som emblemetik. Ingen fjærer, ingen sprett, ingen elastisk overshoot.

02

Formålsdrevet, aldri dekorativ

Hver animasjon forklarer noe: en tilstandsendring, hvor et element kom fra, hva som nå har fokus. Kommuniserer den ikke? Fjern den.

03

Rask og rolig

Grensesnittbevegelse er kort — 120 til 320ms. Merkevaren er autoritativ, ikke oppsiktsvekkende. Hastighet leser som kompetanse.

04

Tilbakeholdenhet

En fokal bevegelse om gangen. Innhold leder — bevegelse støtter og konkurrerer aldri om oppmerksomheten.

02

Timing

Fire varigheter dekker alt. Kortere avstander og enklere endringer bruker kortere tid. Alt over 500ms føles tregt i et grensesnitt.

--motion-fast
120ms
Hover, fokus
--motion-base
200ms
Knapper, toggles
--motion-slow
320ms
Entrances, reveals
--motion-slower
500ms
Store overganger, embleminngang
03

Easing

Fire kurver. Ingen sprett. Geometrisk bevegelse deselererer inn og akselererer ut. Trykk Play for å kjenne hver kurve.

Standard

cubic-bezier(.4, 0, .2, 1)

Elementer som bytter tilstand på stedet — toggles, fargebytte, tab-endringer.

Entrance

cubic-bezier(0, 0, .2, 1)

Elementer som kommer inn i view — modaler, drawere, scroll-reveals.

Exit

cubic-bezier(.4, 0, 1, 1)

Elementer som forlater view — lukkede modaler, tilbakemeldinger som forsvinner.

Emphasis

cubic-bezier(.2, 0, 0, 1)

Store, viktige overganger — embleminngang, heroer, splash-screens.

04

Scroll

Avslør én gang, forsiktig. Innhold fader 16px oppover etter hvert som det entrer — aldri på nytt ved scroll tilbake.

Reglene

  • Fade 16px oppover ved inngang. Aldri re-animer ved scroll ned igjen.
  • Ingen tung parallax — maksimalt en lett differensiell effekt.
  • Stagger er valgfritt og lite — maks 60ms mellom elementer.
  • Alltid respekter prefers-reduced-motion — vis sluttilstand umiddelbart.
Hvert avsnitt på denne siden demonstrerer regelen allerede.
05

Interaksjonstilstander

Interaktive elementer responderer innen 200ms. Hover løfter eller skifter farge — trykket skyver forsiktig ned 2%.

Knapper & lenker
Korthovering
Løfter 4px med skygge
transform + opacity animeres — aldri width, height eller posisjon.

Fokustilstand skal alltid være synlig. Tab gjennom elementene over for å bekrefte.

06

Emblem i bevegelse

Én godkjent inngang. Embleme animerer inn én gang: scale fra 92% med fade, settling på emphasis-kurven.

scale 0.92 → 1 med opacity 0 → 1.
500ms, ease-emphasis. Kun på første paint av hero eller splash — aldri på scroll-loops.

Embleme spinner aldri, looper aldri, forvrenges aldri.
Gapet forblir gapet.
07

Gjør & ikke gjør

Den korte listen. Bruk den som sjekkliste før animasjon i produksjon.

Gjør DO
  • Animer transform og opacity — GPU-billige, aldri reflow.
  • Deselerer entrances, akselerer exits.
  • Hold grensesnittbevegelse under 320ms.
  • Alltid respekter prefers-reduced-motion.
  • Gi hvert interaktivt element en synlig fokustilstand.
Ikke gjør DON'T
  • Ingen sprett, elastisk eller spring overshoot.
  • Ingen loopende eller autospillende dekorativ bevegelse.
  • Ikke animer width, height, top eller left — de janker.
  • Overskrid ikke 500ms for grensesnittoverganger.
  • Ikke flytt mer enn én fokal ting om gangen.
08

Implementering

Legg disse custom properties i det globale stilarket. Denne siden bruker nøyaktig disse verdiene.

/* BlueTec motion tokens */
:root {
  --motion-fast:   120ms;  /* hover, focus */
  --motion-base:   200ms;  /* buttons, toggles */
  --motion-slow:   320ms;  /* entrances, reveals */
  --motion-slower: 500ms;  /* large transitions */

  --ease-standard: cubic-bezier(.4, 0, .2, 1);
  --ease-entrance: cubic-bezier(0,  0, .2, 1);
  --ease-exit:     cubic-bezier(.4, 0,  1, 1);
  --ease-emphasis: cubic-bezier(.2, 0,  0, 1);
}

/* Respect the user's preference — always. */
@media (prefers-reduced-motion: reduce) {
  * {
    transition-duration: 1ms !important;
    animation-duration:  1ms !important;
  }
}
